# AI Image Tool 部署与使用教程

本文档说明如何在 Windows 环境中安装、开发运行、打包部署和使用 AI Image Tool。

## 1. 项目简介

AI Image Tool 是一个 Windows 桌面端 AI 图片生成工具，基于 Electron、React、Vite 和 TypeScript 构建。

主要能力：

- 支持 OpenAI-compatible 图片生成接口。
- 支持 Gemini 原生图片生成接口。
- 支持文字生成图片。
- 支持拖拽或选择参考图。
- 支持本地任务队列和并发生成。
- 支持配置生成尺寸、数量、输出目录、模型和 API Key。
- 生成结果会保存到本地文件，并在界面中展示预览。

## 2. 环境准备

建议环境：

- Windows 10 或 Windows 11。
- Node.js 20 LTS 或更新版本。
- npm。
- Git。

确认环境：

```bash
node -v
npm -v
git --version
```

## 3. 获取项目代码

如果是第一次拉取项目：

```bash
git clone https://github.com/Little-Z7/GenPicsTools.git
cd GenPicsTools
```

如果已经有本地项目目录，进入项目根目录即可。

## 4. 安装依赖

在项目根目录执行：

```bash
npm install
```

依赖会安装到 `node_modules/`。该目录不需要提交到 Git。

## 5. 本地开发运行

执行：

```bash
npm run dev
```

该命令会同时启动：

- Vite 渲染进程开发服务器。
- TypeScript 编译 Electron 主进程代码。
- Electron 桌面应用。

开发时修改前端界面代码后，Vite 会自动刷新；修改主进程或 preload 相关代码后，通常需要重新运行开发命令。

## 6. 验证项目

提交或打包前建议执行：

```bash
npm test -- --run
npm run typecheck
npm run build
```

命令说明：

- `npm test -- --run`：运行 Vitest 测试。
- `npm run typecheck`：运行 TypeScript 类型检查。
- `npm run build`：构建渲染进程和 Electron 主进程代码。

## 7. 打包部署

执行：

```bash
npm run dist
```

打包输出目录：

- `release/win-unpacked/AI Image Tool.exe`
- `release/AI Image Tool 0.1.0.exe`

发布方式：

- 如果要给用户一个单文件启动程序，可以分发 `release/AI Image Tool 0.1.0.exe`。
- 如果要分发目录版，可以复制整个 `release/win-unpacked/` 文件夹。

注意：

- `release/` 是构建产物，已经被 `.gitignore` 忽略，不需要提交到 Git。
- 分发前建议先在本机双击运行打包后的程序，确认启动和生成流程正常。

## 8. 首次使用配置

打开应用后，需要先配置生成服务。

### OpenAI-compatible 配置

适用于兼容 OpenAI 图片接口的服务。

常见配置：

- Provider / Format：`OpenAI Compatible`
- Base URL：例如 `https://api.openai.com/v1`
- Model：例如 `gpt-image-1`
- API Key：填写你的 API Key

接口说明：

- 文生图接口：`{base_url}/images/generations`
- 参考图接口：`{base_url}/images/edits`

Base URL 可以填写根路径 `/v1`，也可以填写完整的 `/images/generations` 或 `/images/edits` 接口地址。

API Key 可以直接填写 `sk-...`，也可以填写 `Bearer sk-...`。

### Gemini 配置

适用于 Gemini 图片生成模型。

常见配置：

- Provider / Format：`Gemini`
- Base URL：`https://generativelanguage.googleapis.com/v1beta`
- Model：例如 `gemini-2.5-flash-image`
- API Key：填写你的 Gemini API Key

接口说明：

- 应用会请求：`{base_url}/models/{model}:generateContent`
- 参考图会以内联图片内容发送给 Gemini。

## 9. 生成图片流程

基本流程：

1. 打开应用。
2. 配置 Provider、Base URL、API Key 和模型。
3. 输入提示词。
4. 选择图片尺寸或比例。
5. 设置生成数量。
6. 选择输出目录。
7. 点击生成。
8. 在任务列表中查看状态。
9. 生成成功后，在结果区域查看图片预览和本地文件路径。

## 10. 使用参考图

支持的参考图格式：

- PNG
- JPG / JPEG
- WebP

添加方式：

- 将图片拖拽到参考图区域。
- 使用文件选择器选择图片。

应用会把参考图复制到本地应用数据目录，因此任务加入队列后，不依赖原始图片继续保留在原位置。

## 11. 任务队列和并发

应用使用本地 SQLite 保存任务队列。

任务状态包括：

- `queued`：排队中。
- `running`：生成中。
- `succeeded`：生成成功。
- `failed`：生成失败。
- `cancelled`：已取消。

并发数用于控制同时执行的任务数量。网络接口不稳定或接口限流时，建议降低并发数。

失败或取消的任务可以重新尝试。

## 12. 本地数据位置

Electron 会把配置、任务记录、日志、参考图副本和输出元数据保存在应用用户数据目录中。

主要数据：

- SQLite 数据库：`app.sqlite`
- 参考图副本目录：`references`
- 生成图片：保存到界面中选择的输出目录

如果需要备份任务记录，请备份应用用户数据目录和图片输出目录。

## 13. Git 提交注意事项

以下内容不应该提交到 Git：

- `node_modules/`
- `.npm-cache/`
- `dist/`
- `dist-electron/`
- `release/`
- `.env`
- 日志文件
- 本地缓存文件
- 编辑器配置和系统临时文件

这些规则已经写入 `.gitignore`。

## 14. 常见问题

### npm install 失败

可以尝试：

```bash
npm cache clean --force
npm install
```

如果网络不稳定，切换 npm 镜像后再安装。

### npm run dev 后应用没有打开

检查：

- 依赖是否安装完成。
- Vite 端口是否被占用。
- Electron 是否安装成功。
- 终端是否有 TypeScript 编译错误。

### 生成失败

检查：

- API Key 是否正确。
- Base URL 是否正确。
- 模型名称是否正确。
- 当前服务是否支持图片生成。
- 参考图格式是否受支持。
- 网络是否可以访问对应接口。
- 是否触发服务商的限流或余额不足。

### 打包后打不开

可以先执行：

```bash
npm run build
```

确认构建无错误后，再执行：

```bash
npm run dist
```

如果目录版可以运行，但便携版不能运行，优先查看 `release/win-unpacked/` 目录版是否完整。

## 15. 推荐发布流程

每次准备发布时：

```bash
npm test -- --run
npm run typecheck
npm run build
npm run dist
```

确认打包结果可运行后，再将 `release/` 中的产物上传到 GitHub Release 或其他分发渠道。

源码仓库只提交源代码、配置、文档和锁文件，不提交构建产物。
