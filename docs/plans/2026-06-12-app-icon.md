# App Icon Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a Windows application icon to the distributable Electron executable.

**Architecture:** Store the icon under `assets/icon.ico` and point electron-builder's Windows config to that file. Keep the icon generated locally so the build does not depend on external artwork or network access.

**Tech Stack:** Electron Builder, Vitest, generated Windows ICO asset.

---

### Task 1: Packaging Config Test

**Files:**
- Create: `src/main/packageConfig.test.ts`
- Modify: `package.json`
- Create: `assets/icon.ico`

**Step 1: Write the failing test**

Check that `package.json` has `build.win.icon` set to `assets/icon.ico` and that the file exists.

**Step 2: Run test to verify it fails**

Run: `npm test -- --run src/main/packageConfig.test.ts`

Expected: FAIL because the icon config is missing.

**Step 3: Add icon and config**

Generate `assets/icon.ico` and update `package.json` with `build.win.icon`.

**Step 4: Run test to verify it passes**

Run: `npm test -- --run src/main/packageConfig.test.ts`

Expected: PASS.

### Task 2: Release Verification

**Files:**
- Existing build outputs.
